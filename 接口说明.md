# 任务 6 接口说明 README

本文档面向两类协作同学：

- **任务 7 / LLM 同学**：需要把自然语言解析成函数调用。
- **前端同学**：后续需要把前端“模板联动配置”下发到 Blender 插件。

当前已实现的任务 6 功能是：在 Blender 插件中选择模板，并批量配置当前 City Generator 场景的道路材质、树木类型、树木参数和人行道资产。

## 当前状态

- Blender 插件内 UI 已可用。
- LLM 函数注册表已预留。
- 前端尚未真实连接 Blender，目前仍是 mock 数据演示。
- 后续前端建议通过后端统一调用插件函数，不建议前端直接调用 Blender Python。

## 一、给任务 7 / LLM 同学

### 可调用函数

函数名：

```text
apply_scene_template
```

作用：

```text
将当前 Blender 激活对象切换到指定场景模板。
模板会批量修改道路材质、树木集合、人行道资产、树木间距和资产间距等参数。
```

前置条件：

```text
Blender 当前激活对象必须已经应用 City_Generator_2.0 Geometry Nodes 修改器。
```

### 函数表获取方式

代码位置：

```text
LLMCityGenerator/function_registry.py
```

调用：

```python
from .function_registry import get_llm_function_specs

function_specs = get_llm_function_specs()
```

返回的函数表可直接提供给 LLM。

### 函数参数 Schema

```json
{
  "name": "apply_scene_template",
  "description": "Apply a predefined city scene detail template to the active Blender City Generator object. The template changes coordinated scene details such as road material, tree type, sidewalk assets, tree spacing, and street-side furniture settings.",
  "parameters": {
    "type": "object",
    "properties": {
      "template_id": {
        "type": "string",
        "enum": ["0", "1", "2"],
        "description": "0 = Waterfront Block, 1 = Commercial Street, 2 = Transit Hub"
      }
    },
    "required": ["template_id"]
  }
}
```

### 模板编号

| template_id | 模板名 | 适合的自然语言 |
| --- | --- | --- |
| `"0"` | Waterfront Block | “切换为滨水街区模板”“使用滨水慢行风格” |
| `"1"` | Commercial Street | “切换为商业街模板”“使用商业街风格” |
| `"2"` | Transit Hub | “切换为交通枢纽模板”“使用枢纽道路风格” |

### LLM 返回调用示例

用户输入：

```text
把当前场景切换成商业街模板。
```

LLM 应返回：

```json
{
  "name": "apply_scene_template",
  "arguments": {
    "template_id": "1"
  }
}
```

### 插件内执行方式

```python
from .function_registry import call_llm_function

result = call_llm_function(
    context,
    "apply_scene_template",
    {"template_id": "1"},
)
```

返回示例：

```json
{
  "template_id": "1",
  "template_name": "Commercial Street",
  "applied": [
    "road_width",
    "lane_amount",
    "sidewalk_scale",
    "tree_distance",
    "tree_min_scale",
    "tree_max_scale",
    "sidewalk_asset_probability",
    "sidewalk_asset_distance",
    "road_material",
    "tree_collection",
    "sidewalk_asset_collection"
  ],
  "warnings": []
}
```

如果 `warnings` 非空，说明部分参数未成功应用，例如资源集合不存在或 Socket 未找到。

### 直接调用底层函数

代码位置：

```text
LLMCityGenerator/template_engine.py
```

函数：

```python
apply_scene_template_function(context, template_id)
```

示例：

```python
from .template_engine import apply_scene_template_function

result = apply_scene_template_function(context, "0")
```

一般情况下，任务 7 同学优先调用 `function_registry.call_llm_function()`，不要直接依赖底层函数。

## 二、给前端同学

### 当前前端状态

当前前端页面已经有“模板联动配置”入口：

```text
src/pages/ProjectsPage.tsx
```

但目前调用的是 mock API，只更新前端内存中的场景状态，没有真实调用 Blender 插件。

当前 mock 调用大致是：

```ts
updateSceneTemplate({
  sceneId: selectedSceneId,
  templateId,
  treeDensity,
  roadWidth,
})
```

### 后续真实接入建议

建议后端提供一个统一的插件函数执行接口，而不是为每个按钮单独写接口。

推荐 API：

```http
POST /api/plugin/execute
Content-Type: application/json
```

请求体：

```json
{
  "projectId": "prj-riverside",
  "sceneId": "scn-river-main",
  "functionName": "apply_scene_template",
  "arguments": {
    "template_id": "1"
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `projectId` | string | 前端项目 ID，用于任务记录和审计 |
| `sceneId` | string | 前端场景 ID，用于任务记录和审计 |
| `functionName` | string | 插件函数名，此处固定为 `apply_scene_template` |
| `arguments.template_id` | string | Blender 插件模板编号，合法值 `"0"`、`"1"`、`"2"` |

响应体建议：

```json
{
  "traceId": "trace-template-001",
  "data": {
    "template_id": "1",
    "template_name": "Commercial Street",
    "applied": ["road_material", "tree_collection", "sidewalk_asset_collection"],
    "warnings": []
  },
  "message": "Template applied successfully."
}
```

### 前端模板 ID 映射

前端 mock 数据目前使用的是语义化模板 ID：

```text
tpl-waterfront
tpl-commercial
tpl-transit
```

Blender 插件使用的是数字模板 ID：

```text
0
1
2
```

后端或前端 realApi 层需要做映射：

```json
{
  "tpl-waterfront": "0",
  "tpl-commercial": "1",
  "tpl-transit": "2"
}
```

前端接入示例：

```ts
const templateIdMap: Record<string, string> = {
  'tpl-waterfront': '0',
  'tpl-commercial': '1',
  'tpl-transit': '2',
};

await api.executePluginFunction({
  projectId: selectedProjectId,
  sceneId: selectedSceneId,
  functionName: 'apply_scene_template',
  arguments: {
    template_id: templateIdMap[templateId],
  },
});
```

### 后端转发到插件的建议实现

后端收到 `/api/plugin/execute` 后，最终应在 Blender Python 环境中调用：

```python
from city_generator.function_registry import call_llm_function

result = call_llm_function(
    blender_context,
    request["functionName"],
    request["arguments"],
)
```

其中 `blender_context` 必须指向当前打开并选中的 City Generator 对象。

## 三、Blender 插件 UI 调用


2. 在 `Scene Template` 下拉框选择模板。

对应 Operator：

```text
cg.apply_scene_template
```

```text
LLMCityGenerator/operators/template_ops.py
```

## 四、当前模板实际修改内容

模板配置文件：

```text
LLMCityGenerator/constants.py
```

当前模板会写入以下 Geometry Nodes Socket：

| 参数 | Socket | 说明 |
| --- | --- | --- |
| `road_width` | `Socket_9` | 道路宽度 |
| `lane_amount` | `Socket_12` | 车道数量 |
| `sidewalk_scale` | `Socket_16` | 人行道尺度 |
| `road_material` | `Socket_57` | 道路主材质 |
| `tree_collection` | `Socket_166` | 树木集合 |
| `tree_distance` | `Socket_171` | 树木间距 |
| `tree_min_scale` | `Socket_182` | 树木最小缩放 |
| `tree_max_scale` | `Socket_183` | 树木最大缩放 |
| `sidewalk_asset_collection` | `Socket_69` | 人行道资产集合 |
| `sidewalk_asset_probability` | `Socket_72` | 人行道资产出现概率 |
| `sidewalk_asset_distance` | `Socket_73` | 人行道资产间距 |

## 五、注意事项

- 本接口要求 Blender 当前激活对象存在 `City_Generator_2.0` 修改器。
- 模板应用会修改当前对象的 Geometry Nodes 参数，不会新建城市对象。
- 当前前端尚未真实连到 Blender，文档中的前端 API 是后续集成建议。
- 如果从 Blender 安装目录运行插件，需确保项目里的 Python 文件已同步到实际插件目录。
